# Gaussian Splat Predict Web Viewer w/Apple's Sharp(ml-sharp)

This repository demonstrates an end-to-end pipeline for generating and visualizing **3D Gaussian Splat scenes** on the web.

Gaussian splat data is generated using **Apple Sharp (ml-sharp)** and rendered in real time in the browser using **Vite** and **Three.js**.

The primary goal of this project is to show that Gaussian splats produced by a native / ML-based pipeline can be **served locally and visualized interactively in a modern web environment** without relying on heavyweight native viewers.

---

## Overview

This project consists of two main stages:

1. **Gaussian Splat Generation**
   Apple Sharp (ml-sharp) is used to generate 3D Gaussian splat data from input imagery. The output is exported as a Gaussian splat-compatible `.ply` file.

2. **Web-based Visualization**
   The generated `.ply` file is loaded locally and rendered in the browser using a Three.js-based Gaussian Splat renderer. Vite is used as the development server and build tool.

---

## Technology Stack

* **Apple Sharp (ml-sharp)** – Gaussian splat generation
* **Three.js** – WebGL-based 3D rendering
* **Vite** – Development server and bundler
* **JavaScript (ES Modules)** – Application logic
* **WebGL2** – Rendering backend

* `main.js` initializes the viewer and loads the splat data.

## Getting Started

### Prerequisites

* Node.js (18+ recommended)
* A modern browser with WebGL2 support

### Installation

```bash
npm install
```

### Run the Viewer

```bash
npm run dev
```

Then open the local URL provided by Vite (typically `http://localhost:5173`).

---

## Usage

* Place your Gaussian splat `.ply` file inside the `models/` directory.
* Update the file path in `main.js` if necessary.
* Start the development server and view the scene directly in the browser.

The viewer supports interactive camera controls such as orbiting, panning, and zooming.

---

## Purpose and Scope

This project is intended as:

* A **proof of concept** for web-based Gaussian splat visualization
* A reference implementation for integrating ML-generated 3D data into web viewers
* A foundation for future work involving GeoAI, 3D reconstruction, and spatial visualization

It is not intended to be a fully featured production renderer, but rather a clean and minimal demonstration of the pipeline.
